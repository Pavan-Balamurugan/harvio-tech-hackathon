# Demo Script

## Suggested 5-Minute Flow

1. Introduce the problem:
   Educational institutions, hostels, and small businesses face phishing and malware risks through links, QR codes, and documents.

2. Show the login:
   Sign in as `admin` to demonstrate role-based access.

3. Show the unified scan form:
   Mention that the same dashboard supports URL, QR content, and document-text analysis.

4. Run a URL scan:
   Use the sample URL button and trigger the scan.
   Point out the verdict, confidence, and risk level.

5. Show scan history:
   Explain that each scan is logged for review and operational visibility.

6. Show alerts:
   Point out that non-safe results generate alert entries and ready channels.

7. Mention fallback readiness:
   Explain that the frontend also includes a demo mode for uninterrupted presentations even if the backend environment is unavailable.

## Suggested URL Example

`http://bit.ly/verify-login?bank=campus-wallet`

## Suggested QR Example

`https://secure-login-example.com/verify-account`

## Suggested Document Example

Filename: `urgent_notice.docm`

Content:

`Urgent action required. Click here to reset password and verify account immediately at http://example-login.com`

## Closing Line

This prototype demonstrates the week-8 milestone: integrated multi-format detection, role-based access, alert visibility, and a reviewable dashboard workflow.
